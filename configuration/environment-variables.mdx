# Environment Variables

LibreChat uses environment variables for configuration. Set these in your `.env` file or system environment.

<Note>
  Copy `.env.example` to `.env` to get started:
  ```bash
  cp .env.example .env
  ```
</Note>

## Server Configuration

### Basic Server Settings

<ParamField path="HOST" type="string" default="localhost">
  Server host address
  
  ```bash
  HOST=localhost
  ```
</ParamField>

<ParamField path="PORT" type="number" default="3080">
  Server port number
  
  ```bash
  PORT=3080
  ```
</ParamField>

<ParamField path="DOMAIN_CLIENT" type="string" required>
  Client-facing domain URL
  
  ```bash
  DOMAIN_CLIENT=http://localhost:3080
  # Production:
  DOMAIN_CLIENT=https://yourdomain.com
  ```
</ParamField>

<ParamField path="DOMAIN_SERVER" type="string" required>
  Server domain URL for backend services
  
  ```bash
  DOMAIN_SERVER=http://localhost:3080
  # Production:
  DOMAIN_SERVER=https://yourdomain.com
  ```
</ParamField>

<ParamField path="NO_INDEX" type="boolean" default="true">
  Add noindex meta tag to prevent search engine indexing
  
  ```bash
  NO_INDEX=true
  ```
</ParamField>

<ParamField path="TRUST_PROXY" type="number" default="1">
  Number of hops from the Express application. Set to 0 if no reverse proxy.
  
  ```bash
  # Behind 1 reverse proxy (default)
  TRUST_PROXY=1
  
  # Behind 2 reverse proxies
  TRUST_PROXY=2
  
  # No reverse proxy
  TRUST_PROXY=0
  ```
</ParamField>

<ParamField path="MIN_PASSWORD_LENGTH" type="number" default="8">
  Minimum password length for user authentication
  
  <Tip>When using LDAP authentication, set this to 1 to bypass local password validation</Tip>
  
  ```bash
  MIN_PASSWORD_LENGTH=8
  # For LDAP:
  MIN_PASSWORD_LENGTH=1
  ```
</ParamField>

<ParamField path="CONTINUE_ON_UNCAUGHT_EXCEPTION" type="boolean" default="false">
  Continue running after encountering uncaught exceptions
  
  <Warning>Not recommended for production unless necessary</Warning>
  
  ```bash
  CONTINUE_ON_UNCAUGHT_EXCEPTION=false
  ```
</ParamField>

### Node.js Options

<ParamField path="NODE_MAX_OLD_SPACE_SIZE" type="number" default="6144">
  Build argument for Docker/CI workflows (NOT used by Node.js directly)
  
  To configure Node.js memory at runtime, use NODE_OPTIONS:
  
  ```bash
  NODE_MAX_OLD_SPACE_SIZE=6144
  
  # For runtime memory configuration:
  NODE_OPTIONS="--max-old-space-size=6144"
  ```
</ParamField>

## Database Configuration

### MongoDB

<ParamField path="MONGO_URI" type="string" required>
  MongoDB connection string
  
  ```bash
  MONGO_URI=mongodb://127.0.0.1:27017/LibreChat
  
  # With authentication:
  MONGO_URI=mongodb://username:password@127.0.0.1:27017/LibreChat
  
  # MongoDB Atlas:
  MONGO_URI=mongodb+srv://username:password@cluster.mongodb.net/LibreChat
  ```
</ParamField>

<ParamField path="MONGO_MAX_POOL_SIZE" type="number">
  Maximum number of connections in the connection pool
  
  ```bash
  MONGO_MAX_POOL_SIZE=10
  ```
</ParamField>

<ParamField path="MONGO_MIN_POOL_SIZE" type="number">
  Minimum number of connections in the connection pool
  
  ```bash
  MONGO_MIN_POOL_SIZE=2
  ```
</ParamField>

<ParamField path="MONGO_MAX_CONNECTING" type="number">
  Maximum number of connections being established concurrently
  
  ```bash
  MONGO_MAX_CONNECTING=5
  ```
</ParamField>

<ParamField path="MONGO_MAX_IDLE_TIME_MS" type="number">
  Maximum milliseconds a connection can remain idle before being closed
  
  ```bash
  MONGO_MAX_IDLE_TIME_MS=60000
  ```
</ParamField>

<ParamField path="MONGO_WAIT_QUEUE_TIMEOUT_MS" type="number">
  Maximum time in milliseconds to wait for a connection
  
  ```bash
  MONGO_WAIT_QUEUE_TIMEOUT_MS=5000
  ```
</ParamField>

<ParamField path="MONGO_AUTO_INDEX" type="boolean" default="true">
  Enable automatic index creation for all models
  
  ```bash
  MONGO_AUTO_INDEX=true
  ```
</ParamField>

<ParamField path="MONGO_AUTO_CREATE" type="boolean" default="true">
  Enable automatic collection creation
  
  ```bash
  MONGO_AUTO_CREATE=true
  ```
</ParamField>

<ParamField path="LANGFUSE_PUBLIC_KEY" type="string">

<ParamField path="LANGFUSE_SECRET_KEY" type="string">

  ```bash
  LANGFUSE_SECRET_KEY=sk-lf-xxx
  ```
</ParamField>

<ParamField path="LANGFUSE_BASE_URL" type="string">
  Langfuse API base URL (optional, for self-hosted)
  
  ```bash
  LANGFUSE_BASE_URL=https://your-langfuse-instance.com
  ```

</ParamField>

## Security

### JWT Secrets

<ParamField path="JWT_SECRET" type="string" required>
  Secret key for JWT token signing
  
  <Warning>Generate a secure random string and keep it secret!</Warning>
  
  ```bash
  # Generate with:
  openssl rand -hex 32
  
  JWT_SECRET=16f8c0ef4a5d391b26034086c628469d3f9f497f08163ab9b40137092f2909ef
  ```
</ParamField>

<ParamField path="JWT_REFRESH_SECRET" type="string" required>
  Secret key for refresh token signing (must be different from JWT_SECRET)
  
  ```bash
  # Generate with:
  openssl rand -hex 32
  
  JWT_REFRESH_SECRET=eaa5191f2914e30b9387fd84e254e4ba6fc51b4654968a9b0803b456a54b8418
  ```
</ParamField>

### Credential Encryption

<ParamField path="CREDS_KEY" type="string" required>
  Encryption key for stored credentials (32 bytes hex)
  
  ```bash
  # Generate with:
  openssl rand -hex 32
  
  CREDS_KEY=f34be427ebb29de8d88c107a71546019685ed8b241d8f2ed00c3df97ad2566f0
  ```
</ParamField>

<ParamField path="CREDS_IV" type="string" required>
  Initialization vector for credential encryption (16 bytes hex)
  
  ```bash
  # Generate with:
  openssl rand -hex 16
  
  CREDS_IV=e2341419ec3dd3d19b13a1a87fafcbfb
  ```
</ParamField>

### Session Configuration

<ParamField path="SESSION_EXPIRY" type="number" default="900000">
  Session expiry time in milliseconds
  
  ```bash
  # Default: 15 minutes
  SESSION_EXPIRY=1000 * 60 * 15
  
  # 1 hour:
  SESSION_EXPIRY=3600000
  ```
</ParamField>

<ParamField path="REFRESH_TOKEN_EXPIRY" type="number" default="604800000">
  Refresh token expiry time in milliseconds
  
  ```bash
  # Default: 7 days
  REFRESH_TOKEN_EXPIRY=(1000 * 60 * 60 * 24) * 7
  
  # 30 days:
  REFRESH_TOKEN_EXPIRY=2592000000
  ```
</ParamField>
