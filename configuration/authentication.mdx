# Authentication Configuration

LibreChat supports multiple authentication methods including email/password, OAuth2 social providers, OpenID Connect, SAML, and LDAP.

## Registration and Login Settings

### Basic Authentication Controls

<ParamField path="ALLOW_EMAIL_LOGIN" type="boolean" default="true">
  Enable email/password authentication
  
  ```bash
  ALLOW_EMAIL_LOGIN=true
  ```
</ParamField>

<ParamField path="ALLOW_REGISTRATION" type="boolean" default="true">
  Allow new user registration
  
  ```bash
  ALLOW_REGISTRATION=true
  # Production (invite-only):
  ALLOW_REGISTRATION=false
  ```
</ParamField>

<ParamField path="ALLOW_SOCIAL_LOGIN" type="boolean" default="false">
  Allow OAuth social login for existing users
  
  ```bash
  ALLOW_SOCIAL_LOGIN=false
  ```
</ParamField>

<ParamField path="ALLOW_SOCIAL_REGISTRATION" type="boolean" default="false">
  Allow new user registration via social OAuth providers
  
  ```bash
  ALLOW_SOCIAL_REGISTRATION=false
  ```
</ParamField>

<ParamField path="ALLOW_PASSWORD_RESET" type="boolean" default="false">
  Enable password reset functionality (requires email service)
  
  ```bash
  ALLOW_PASSWORD_RESET=false
  ```
</ParamField>

<ParamField path="ALLOW_ACCOUNT_DELETION" type="boolean" default="true">
  Allow users to delete their own accounts
  
  ```bash
  ALLOW_ACCOUNT_DELETION=true
  ```
</ParamField>

<ParamField path="ALLOW_UNVERIFIED_EMAIL_LOGIN" type="boolean" default="true">
  Allow login with unverified email addresses
  
  ```bash
  ALLOW_UNVERIFIED_EMAIL_LOGIN=true
  # Production:
  ALLOW_UNVERIFIED_EMAIL_LOGIN=false
  ```
</ParamField>

### Domain Restrictions

Restrict registration to specific email domains using `librechat.yaml`:

```yaml librechat.yaml
registration:
  socialLogins: ['github', 'google', 'discord', 'openid']
  allowedDomains:
    - "company.com"
    - "partner.org"
```

## OAuth2 Social Login

LibreChat supports multiple OAuth2 providers for social login.

### Google OAuth

<Steps>
  <Step title="Create OAuth App">
    1. Go to [Google Cloud Console](https://console.cloud.google.com/)
    2. Create a new project or select existing
    3. Enable Google+ API
    4. Go to **Credentials** > **Create Credentials** > **OAuth client ID**
    5. Select **Web application**
  </Step>
  
  <Step title="Configure Redirect URIs">
    Add authorized redirect URI:
    ```
    https://yourdomain.com/oauth/google/callback
    ```
  </Step>
  
  <Step title="Set Environment Variables">
    ```bash .env
    GOOGLE_CLIENT_ID=your-client-id.apps.googleusercontent.com
    GOOGLE_CLIENT_SECRET=your-client-secret
    GOOGLE_CALLBACK_URL=/oauth/google/callback
    
    # Enable social login
    ALLOW_SOCIAL_LOGIN=true
    ALLOW_SOCIAL_REGISTRATION=true
    ```
  </Step>
  
  <Step title="Enable in YAML">
    ```yaml librechat.yaml
    registration:
      socialLogins: ['google']
    ```
  </Step>
</Steps>

<ParamField path="GOOGLE_CLIENT_ID" type="string">
  Google OAuth client ID
</ParamField>

<ParamField path="GOOGLE_CLIENT_SECRET" type="string">
  Google OAuth client secret
</ParamField>

<ParamField path="GOOGLE_CALLBACK_URL" type="string" default="/oauth/google/callback">
  OAuth callback URL path
</ParamField>

### GitHub OAuth

<Steps>
  <Step title="Create OAuth App">
    1. Go to GitHub **Settings** > **Developer settings** > **OAuth Apps**
    2. Click **New OAuth App**
    3. Fill in application details
  </Step>
  
  <Step title="Configure Callback URL">
    Set the callback URL:
    ```
    https://yourdomain.com/oauth/github/callback
    ```
  </Step>
  
  <Step title="Set Environment Variables">
    ```bash .env
    GITHUB_CLIENT_ID=your-client-id
    GITHUB_CLIENT_SECRET=your-client-secret
    GITHUB_CALLBACK_URL=/oauth/github/callback
    
    # For GitHub Enterprise:
    # GITHUB_ENTERPRISE_BASE_URL=https://github.enterprise.com
    # GITHUB_ENTERPRISE_USER_AGENT=LibreChat
    ```
  </Step>
</Steps>

## OpenID Connect

OpenID Connect (OIDC) provides SSO integration with enterprise identity providers like Azure AD, Keycloak, Okta, and Auth0.

### Basic Configuration

<ParamField path="OPENID_CLIENT_ID" type="string" required>
  OpenID client ID from your identity provider
  
  ```bash
  OPENID_CLIENT_ID=your-client-id
  ```
</ParamField>

<ParamField path="OPENID_CLIENT_SECRET" type="string" required>
  OpenID client secret
  
  ```bash
  OPENID_CLIENT_SECRET=your-client-secret
  ```
</ParamField>

<ParamField path="OPENID_ISSUER" type="string" required>
  OpenID provider issuer URL
  
  ```bash
  # Azure AD:
  OPENID_ISSUER=https://login.microsoftonline.com/{tenant-id}/v2.0
  
  # Keycloak:
  OPENID_ISSUER=https://keycloak.example.com/realms/myrealm
  
  # Okta:
  OPENID_ISSUER=https://dev-xxxxx.okta.com
  
  # Auth0:
  OPENID_ISSUER=https://your-tenant.auth0.com
  ```
</ParamField>

<ParamField path="OPENID_SESSION_SECRET" type="string" required>
  Secret for session encryption
  
  ```bash
  # Generate with: openssl rand -hex 32
  OPENID_SESSION_SECRET=your-session-secret
  ```
</ParamField>

<ParamField path="OPENID_SCOPE" type="string" default="openid profile email">
  OAuth scopes to request
  
  ```bash
  OPENID_SCOPE="openid profile email"
  ```
</ParamField>

<ParamField path="OPENID_CALLBACK_URL" type="string" default="/oauth/openid/callback">
  Callback URL path
  
  ```bash
  OPENID_CALLBACK_URL=/oauth/openid/callback
  ```
</ParamField>

### User Attribute Mapping

<ParamField path="OPENID_USERNAME_CLAIM" type="string">
  Claim to use for username (e.g., `preferred_username`, `name`)
  
  ```bash
  OPENID_USERNAME_CLAIM=preferred_username
  ```
</ParamField>

<ParamField path="OPENID_NAME_CLAIM" type="string">
  Claim to use for display name
  
  ```bash
  OPENID_NAME_CLAIM=name
  ```
</ParamField>

<ParamField path="OPENID_EMAIL_CLAIM" type="string">
  Claim to use for email/identifier
  
  When not set, defaults to: `email` -> `preferred_username` -> `upn`
  
  ```bash
  # For Entra ID:
  OPENID_EMAIL_CLAIM=upn
  ```
</ParamField>

## LDAP

LDAP authentication integrates with Active Directory and other LDAP servers.

<ParamField path="LDAP_URL" type="string" required>
  LDAP server URL
  
  ```bash
  LDAP_URL=ldap://ldap.example.com:389
  # Or with TLS:
  LDAP_URL=ldaps://ldap.example.com:636
  ```
</ParamField>

<ParamField path="LDAP_BIND_DN" type="string">
  Bind DN for LDAP authentication
  
  ```bash
  LDAP_BIND_DN=cn=admin,dc=example,dc=com
  ```
</ParamField>

<ParamField path="LDAP_BIND_CREDENTIALS" type="string">
  Bind password for LDAP authentication
  
  ```bash
  LDAP_BIND_CREDENTIALS=your-bind-password
  ```
</ParamField>

<ParamField path="LDAP_USER_SEARCH_BASE" type="string" required>
  Base DN for user searches
  
  ```bash
  LDAP_USER_SEARCH_BASE=ou=users,dc=example,dc=com
  ```
</ParamField>

<ParamField path="LDAP_SEARCH_FILTER" type="string" default="mail={{username}}">
  LDAP search filter template
  
  ```bash
  LDAP_SEARCH_FILTER=mail={{username}}
  # Active Directory:
  LDAP_SEARCH_FILTER=sAMAccountName={{username}}
  ```
</ParamField>

  ```bash
  LDAP_CA_CERT_PATH=/path/to/ca-cert.pem
  ```
</ParamField>

<ParamField path="LDAP_TLS_REJECT_UNAUTHORIZED" type="boolean" default="true">
  Reject unauthorized TLS certificates
  
  ```bash
  LDAP_TLS_REJECT_UNAUTHORIZED=true
  ```
</ParamField>

<ParamField path="LDAP_STARTTLS" type="boolean" default="false">
  Use STARTTLS for encryption
  
  ```bash
  LDAP_STARTTLS=false
  ```
</ParamField>

### LDAP Attribute Mapping

<ParamField path="LDAP_ID" type="string">
  LDAP attribute for user ID (defaults to `uid` or `sAMAccountName`)
  
  ```bash
  LDAP_ID=employeeNumber
  ```
</ParamField>

<ParamField path="LDAP_USERNAME" type="string">
  LDAP attribute for username (defaults to `givenName` or `mail`)
  
  ```bash
  LDAP_USERNAME=sAMAccountName
  ```
</ParamField>

<ParamField path="LDAP_EMAIL" type="string">
  LDAP attribute for email (defaults to `mail`)
  
  ```bash
  LDAP_EMAIL=mail
  ```
</ParamField>

<ParamField path="LDAP_FULL_NAME" type="string">
  Comma-separated list of LDAP attributes for full name
  
  ```bash
  LDAP_FULL_NAME=givenName,sn
  # Or use a single attribute:
  LDAP_FULL_NAME=displayName
  ```
</ParamField>

<ParamField path="LDAP_LOGIN_USES_USERNAME" type="boolean" default="false">
  Use username field instead of email for login
  
  ```bash
  LDAP_LOGIN_USES_USERNAME=true
  ```
</ParamField>

<Tip>
  **LDAP Password Validation**: When using LDAP, set `MIN_PASSWORD_LENGTH=1` to bypass local password validation, as LDAP servers handle their own password policies.
  
  ```bash
  MIN_PASSWORD_LENGTH=1
  ```
</Tip>

## Troubleshooting

<AccordionGroup>
  <Accordion title="OAuth callback errors">
    - Verify callback URLs match exactly in provider settings and .env
    - Check that `DOMAIN_CLIENT` and `DOMAIN_SERVER` are correctly configured
    - Ensure HTTPS is used in production
    - Verify OAuth app is enabled and approved
  </Accordion>
  
  <Accordion title="OpenID token errors">
    - Check issuer URL is correct and accessible
    - Verify client ID and secret are correct
    - Ensure required scopes are granted in identity provider
    - Check token expiration and refresh token settings
    - Review JWKS cache settings if using custom configuration
  </Accordion>
  
  <Accordion title="LDAP connection issues">
    - Verify LDAP URL is correct and server is accessible
    - Check bind DN and credentials are valid
    - Test LDAP connection using ldapsearch or similar tool
    - Verify user search base and search filter are correct
    - For TLS issues, check CA certificate path and hostname
  </Accordion>
  
  <Accordion title="Role-based access not working">
    - Verify role claims are present in tokens (decode JWT)
    - Check role parameter path matches token structure
    - Ensure token kind (id_token vs access_token) is correct
    - Verify role names match exactly (case-sensitive)
  </Accordion>
</AccordionGroup>
