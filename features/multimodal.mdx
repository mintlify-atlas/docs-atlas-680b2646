## Uploading Files

### Supported File Types

<Tabs>
  <Tab title="Images">
    **Image Formats:**
    - JPEG/JPG
    - PNG
    - GIF
    - WebP
    - BMP
    - SVG (converted to raster)
    
    ```yaml librechat.yaml
    fileConfig:
      endpoints:
        openAI:
          supportedMimeTypes:
            - "image/.*"
    ```
  </Tab>
  
  <Tab title="Documents">
    **Document Formats:**
    - PDF
    - TXT
    - Markdown
    - CSV
    - JSON
    - Code files (.js, .py, .java, etc.)
    
    ```yaml librechat.yaml
    fileConfig:
      endpoints:
        openAI:
          supportedMimeTypes:
            - "application/pdf"
            - "text/.*"
    ```
  </Tab>
  
  <Tab title="Data Files">
    **Data Formats:**
    - CSV
    - Excel (.xlsx, .xls)
    - JSON
    - XML
    
    Useful for data analysis with Code Interpreter.
  </Tab>
</Tabs>

### File Upload Configuration

```yaml librechat.yaml
fileConfig:
  endpoints:
    openAI:
      fileLimit: 10  # Max files per request
      fileSizeLimit: 20  # MB per file
      totalSizeLimit: 100  # MB total per request
      supportedMimeTypes:
        - "image/.*"
        - "application/pdf"
    
    anthropic:
      fileLimit: 5
      fileSizeLimit: 10
      totalSizeLimit: 50
      supportedMimeTypes:
        - "image/.*"
        - "application/pdf"
  
  # Global server limit
  serverFileSizeLimit: 100  # MB
  
  # Avatar images
  avatarSizeLimit: 2  # MB
```

### Client-Side Image Resizing

Prevent upload errors by resizing large images:

```yaml librechat.yaml
fileConfig:
  clientImageResize:
    enabled: true
    maxWidth: 1900
    maxHeight: 1900
    quality: 0.92  # JPEG quality (0.0-1.0)
```

## Using Multimodal Features

### Image Analysis

Upload images and ask questions:

```
User: [Uploads screenshot of error message]
What's causing this error and how can I fix it?

Assistant: Looking at your screenshot, the error is a 
"TypeError: Cannot read property 'map' of undefined". This 
occurs on line 42 where you're trying to map over `data.items`.

The issue is that `data.items` is undefined, likely because:
1. The API response doesn't include an `items` property
2. The data hasn't loaded yet when the component renders

Fix:
```javascript
{data?.items?.map(item => ...)}
// or
{(data.items || []).map(item => ...)}
```
```

### Image Processing with Code

Combine vision models with code execution:

```python
from PIL import Image
import numpy as np
import matplotlib.pyplot as plt

# Load the uploaded image
img = Image.open('photo.jpg')
img_array = np.array(img)

# Analyze dimensions
height, width, channels = img_array.shape
print(f"Image size: {width}x{height}, Channels: {channels}")

# Calculate average color
avg_color = img_array.mean(axis=(0, 1))
print(f"Average RGB: {avg_color}")

# Create histogram
plt.figure(figsize=(10, 4))
for i, color in enumerate(['red', 'green', 'blue']):
    plt.subplot(1, 3, i+1)
    plt.hist(img_array[:,:,i].ravel(), bins=256, color=color, alpha=0.7)
    plt.title(f'{color.capitalize()} Channel')
plt.tight_layout()
plt.savefig('color_histogram.png')
```

### File Storage Strategies

Configure where files are stored:

```yaml librechat.yaml
# Single strategy for all files
fileStrategy: "s3"

# Or granular control
fileStrategy:
  avatar: "s3"      # User/agent avatars
  image: "firebase" # Chat images
  document: "local" # Documents
```

**Available strategies:**
- `local`: Local file system
- `s3`: AWS S3 or compatible
- `firebase`: Firebase Storage

## Best Practices

<Tip>
  **Image Quality**: Higher resolution images provide better analysis but take longer to process
</Tip>

<Tip>
  **Multiple Images**: Upload multiple angles or views for comprehensive analysis
</Tip>

<Tip>
  **Clear Questions**: Be specific about what you want to know from the image/document
</Tip>

<Warning>
  **File Sizes**: Large files may hit upload limits or timeout - compress when possible
</Warning>

<Info>
  **Privacy**: Uploaded files are processed by AI providers - don't upload sensitive data without proper security
</Info>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Images Not Processing">
    **Check:**
    - Model supports vision (gpt-4o, claude-4.5, etc.)
    - File size within limits
    - Correct MIME type (image/*)
    - Image format is supported
  </Accordion>
  
  <Accordion title="Poor Quality Analysis">
    **Improve:**
    - Upload higher resolution images
    - Ensure good lighting and clarity
    - Crop to relevant content
    - Use specific prompts
  </Accordion>
  
  <Accordion title="PDF Not Working">
    **Solutions:**
    - Use document-supported provider
    - Check file size limits
    - Verify PDF is not corrupted
    - Try splitting large PDFs
  </Accordion>
</AccordionGroup>
